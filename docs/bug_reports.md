## BUG-001: Форма checkout не готова к взаимодействию сразу после перехода

**Статус:** Исследован / Workaround найден  
**Приоритет:** Высокий  
**Компонент:** Оформление заказа — шаг 1 (форма с именем)  
**Воспроизводимость:** Стабильно при быстром переходе со страницы корзины  

### Шаги для воспроизведения
1. Авторизоваться как standard_user
2. Добавить товар в корзину
3. Перейти в корзину → нажать Checkout
4. Немедленно попытаться ввести данные в поле First Name

### Ожидаемый результат
Поля формы доступны для ввода сразу после загрузки страницы

### Фактический результат
Selenium выбрасывает `ElementNotInteractableException` — элемент найден в DOM,
но ещё не доступен для взаимодействия

### Причина
Страница рендерится не мгновенно — элементы появляются в DOM раньше,
чем становятся интерактивными

### Решение
Добавлено явное ожидание `WebDriverWait + visibility_of_element_located`
перед заполнением формы. После исправления тест TC-008 стабильно проходит.

### Вывод для тестировщика
Всегда использовать явные ожидания вместо `implicitly_wait` для форм,
которые загружаются после навигации

## BUG-002: Сообщение об ошибке не скрывается при повторном вводе

**Статус:** Open  
**Приоритет:** Низкий  
**Компонент:** Форма входа  
**Воспроизводимость:** Всегда  

### Шаги для воспроизведения
1. Открыть https://www.saucedemo.com
2. Ввести неверный логин и пароль → появляется красная ошибка
3. Начать исправлять текст в поле Username

### Ожидаемый результат
Сообщение об ошибке скрывается когда пользователь начинает редактировать поля

### Фактический результат
Сообщение об ошибке остаётся на экране до тех пор пока пользователь
вручную не нажмёт крестик (×) в блоке ошибки

### Влияние
UX-дефект: пользователь видит ошибку даже когда уже исправляет данные,
что создаёт путаницу

### Связанные тесты
TC-002, TC-003

## BUG-003: Нет ограничения на длину поля Postal Code

**Статус:** Open  
**Приоритет:** Средний  
**Компонент:** Форма оформления заказа  
**Воспроизводимость:** Всегда  

### Шаги для воспроизведения
1. Пройти до формы оформления заказа
2. Ввести в поле Postal Code строку из 100+ символов
3. Нажать Continue

### Ожидаемый результат
Поле ограничено по длине или показывает ошибку валидации

### Фактический результат
Форма принимает любую длину без ошибок и переходит к следующему шагу

### Влияние
Потенциальная уязвимость — отсутствие валидации на фронтенде
